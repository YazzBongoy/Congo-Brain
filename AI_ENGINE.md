# AI Engine – IAGov

## Purpose
The **AI Engine** provides intelligent agents that assist policymakers :
- **Policy Forecasting** – Predict economic outcomes of a proposal using time‑series models.
- **Corruption Detection** – Analyse transaction logs, procurement contracts, and communication metadata to flag anomalous patterns.
- **RAG‑Based Knowledge Retrieval** – Retrieve relevant legislation, past decisions, and research papers from a vector store.
- **Digital Twin Simulation** – Run an agent‑based model of the national economy to test “what‑if” scenarios.

## Core Services (FastAPI endpoints)
| Endpoint | Method | Description |
|----------|--------|-------------|
| `/ai/forecast` | POST | Takes a policy description, returns projected macro‑economic indicators (GDP, inflation, unemployment). |
| `/ai/corruption/detect` | POST | Accepts a CSV of procurement transactions, returns risk scores and flagged entries. |
| `/ai/rag/query` | POST | Query the knowledge base; returns top‑k passages with citations. |
| `/ai/digital‑twin/simulate` | POST | Runs a multi‑agent simulation for a specified horizon (e.g., 5 years). |

## Example – Corruption Detection (Python)
```python
import pandas as pd, requests, json
# Load procurement CSV
proc = pd.read_csv('data/procurement_2023.csv')
# Send to AI service
resp = requests.post('https://api.iagov.example.com/ai/corruption/detect', json={"records": proc.to_dict(orient='records')})
result = resp.json()
print('Flagged high‑risk contracts:', len([r for r in result['flags'] if r['risk'] > 0.8]))
```

## Model Choices
- **LLM**: Claude‑3.5 Sonnet for RAG and policy summarisation (high fidelity, low latency).
- **Time‑Series**: Prophet + XGBoost ensemble for macro‑forecasting.
- **Corruption ML**: Gradient Boosted Trees trained on labelled procurement data (public‑procurement‑datasets).
- **Digital Twin**: Mesa (Python ABM) with agents representing households, firms, ministries.

## Deployment
The AI Engine runs in its own **Kubernetes namespace** (`ai-engine`). It pulls the LLM model via the **Claude API** (requires API key stored in Vault). Workers are scaled horizontally based on request latency.
