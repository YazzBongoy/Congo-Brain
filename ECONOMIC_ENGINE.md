# Economic Engine – IAGov

## Purpose
The Economic Engine computes the **Surplus National Net (SNN)** and evaluates policy proposals using mathematical optimisation, simulation, and AI‑augmented forecasts.

## Core Concepts
- **Consumer Surplus (CS)** – area under the demand curve above the market price.
- **Producer Surplus (PS)** – area above the supply curve below the market price.
- **Government Revenue (GR)** – taxes, royalties, and fees collected.
- **Natural Resource Value (NRV)** – added value from local transformation of resources (e.g., cobalt → batteries).
- **Dead‑Weight Loss (DWL)** – losses from corruption, monopolies, administrative costs.
- **Environmental Cost (EC)** – quantified externalities (CO₂, deforestation, water use).

## SNN Formula
```
SNN = Σ_i (CS_i + PS_i + GR_i + NRV_i) – Σ_j (DWL_j + EC_j)
```
- `i` iterates over all projects / policies.
- `j` iterates over all loss sources.

## Optimisation Models
| Model | Description | Solver |
|-------|-------------|--------|
| **Linear Programming (LP)** | Allocation of a fixed budget across projects with linear returns. | HiGHS / GLPK |
| **Mixed‑Integer Linear Programming (MILP)** | Discrete decisions (e.g., building a dam vs. a road). | CBC, Gurobi (if licensed) |
| **Stochastic Programming** | Handles uncertainty in commodity prices, exchange rates. | Pyomo + Monte‑Carlo |
| **Multi‑Objective Optimisation** | Simultaneously maximises CS, PS, NRV, minimises DWL & EC. | NSGA‑II (DEAP) |
| **Game Theory Models** | Strategic interactions between ministries or private firms. | Nashpy |

## API Endpoints (excerpt)
- `POST /api/economic/optimise` – Submit a project portfolio and constraints, receive optimal allocation.
- `GET /api/economic/metrics/{project_id}` – Retrieve CS, PS, GR, NRV, DWL, EC for a given project.
- `POST /api/economic/simulate` – Run Monte‑Carlo simulations for budget scenarios.

## Data Sources
- **Fiscal Data** – Ministry of Finance APIs (taxes, expenditures).
- **Resource Extraction** – Mining ministry CSV exports, satellite‑derived production.
- **Market Prices** – Bloomberg, World Bank commodity price feeds.
- **Corruption Indices** – Transparency International, internal audit logs.

## Integration
The engine is packaged as a **Python micro‑service** (`economic-engine`) exposing a **FastAPI** REST interface. Heavy calculations are off‑loaded to a **Celery** worker pool using **Redis** as broker.

## Example Usage (Python client)
```python
import requests, json
payload = {
    "budget": 10_000_000_000,
    "projects": [
        {"id": "road-1", "roi": 0.12, "cs": 3.5, "ps": 2.1, "nr": 0.4, "dwl": 0.1, "ec": 0.05},
        {"id": "cobalt-plant", "roi": 0.25, "cs": 1.2, "ps": 4.8, "nr": 2.5, "dwl": 0.2, "ec": 0.1}
    ]
}
resp = requests.post('https://api.iagov.example.com/economic/optimise', json=payload)
print(json.dumps(resp.json(), indent=2))
```
